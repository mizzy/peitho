# Markdown Images

## Summary

Resolves #117.
Resolves #106.

This PR adds Markdown image support for local deck-relative paths:

```markdown
![Architecture diagram](img/arch.png)
```

Images are parsed as typed `FragmentKind::Image { alt, src }`, mapped only to `accepts="image"` slots, resolved into hashed distribution assets, rendered as `<img>`, listed in `manifest.json`, copied into both `dist/assets/` and the present cache, and validated during publish.

## Design Notes

- `RawImagePath` and `ResolvedImagePath` are separate newtypes. The parser produces raw deck-relative paths; only the resolver can produce resolved `assets/<hash>-<basename>` paths.
- `FragmentKind<S>` / `SourceFragment<S>` and `Deck<Checked<S>>` carry the image path state through the typestate pipeline.
- `render_deck` accepts `Deck<Checked<ResolvedImagePath>>`, and compile-fail doctests cover that `Deck<Checked<RawImagePath>>` cannot reach rendering.
- Core owns parsing, mapping, checking, rendering, manifest shape, and resolver traversal. The CLI injects the resolver that performs filesystem I/O, hashing, deduplication, and asset copy.
- The MVP allow-list is `png`, `jpg`, `jpeg`, `gif`, `webp`. SVG is deliberately rejected until policy is decided.
- Raw paths reject remote URLs, absolute paths, parent-directory escapes, empty paths, backslashes, query strings, fragments, and unsupported extensions at parse time.

## Security And Failure Handling

- Build resolver canonicalizes the deck directory and image source to reject symlink escapes outside the deck.
- Publish validation canonicalizes manifest slide and image refs to reject symlink escapes outside `dist/`.
- CLI resolver converts missing, unreadable, and escaped image paths into `BuildError(ErrorKind::Asset)` with deck-relative messages, line context, and help.
- `dist/assets/` and present-cache `assets/` are recreated on every emit so stale image copies do not survive a successful rebuild.
- Asset names use the first 16 hex chars of SHA-256 plus the source basename: `assets/<hash>-<basename>`.

## Manifest And Compatibility

- `manifest.json` now includes `images: [{ "src": "assets/..." }]`.
- The schema change is additive: Rust deserialization defaults missing `images` to `[]`.
- New builds always serialize `images`, including `images: []` for decks without images.
- TypeScript bindings include `ManifestImage` and `Manifest.images`; presenter runtime remains compatible with older manifests that omit `images`.

## Known Limitations

- Mixed paragraphs such as `text ![alt](x.png) text` are still explicit parse errors.
- Multiple images in one paragraph are still explicit parse errors; multiple image paragraphs are supported subject to slot arity.
- SVG remains unsupported until sanitization/opaque-asset policy is decided.
- Image size hints are not part of Markdown syntax yet; use CSS.
- Layouts with multiple `accepts="image"` slots are ambiguous unless a future slot-selection syntax is designed.
- Alt text is flattened as plain text; richer Markdown inside alt is not preserved.
- `--watch` does not dynamically add referenced image files to the watcher yet. A rebuild clears stale assets, but image-file changes do not themselves trigger rebuilds.
- There is still a TOCTOU window between hashing an image and copying it during emit.
