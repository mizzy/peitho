# ゼロコンフィグ規約（デッキ隣接のlayouts/・css/自動検出）

## 著者判断（2026-07-03、Issue #17の方針）

デッキ別リポジトリ運用は、まず**規約決め打ちのゼロコンフィグ**でいく。peitho.tomlはカスタマイズの必要性が出てから検討する。

## 規約

- `--layouts`未指定のとき、**デッキファイルと同じディレクトリ**に`layouts/`があればそれを使う。無ければ内蔵レイアウト
- `--css`未指定のとき、同様に`css/`があれば使う。無ければ内蔵baseテーマ
- フラグ明示は常に規約より優先
- 規約ディレクトリが存在するのに対象ファイルが空の場合は**ビルドエラー**（作った以上は意図があるはず。内蔵への黙ったフォールバックはしない）
- 基準はカレントディレクトリではなくデッキの隣（デッキrepoをどこから叩いても同じ結果になる）

これで典型的なデッキrepoは `deck.md` + `layouts/` + `css/` を置くだけになり、`peitho present deck.md`で動く。

## 検証

- 単体: フラグ優先/規約検出/どちらも無ければ内蔵、の解決順
- 統合: 隣接layouts/+css/があるデッキをフラグ無しでビルド→規約側が使われる。無いデッキ→内蔵
- E2E: examples/keynoteをフラグ無しでbuild/present（examplesは前PRでこの構造になっている）
- --watch: 規約解決後のディレクトリを監視対象に含める（起動時解決。起動後にディレクトリを新設した場合はwatch再起動が必要、は許容）
